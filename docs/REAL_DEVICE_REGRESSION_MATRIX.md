# Real Device Regression Matrix (Android)

目标：覆盖“是否算重构完成”最关键链路，优先验证端到端稳定性与可回归性。

验收总标准：
- 无闪退
- 任务状态流转正确（pending -> processing -> completed/failed）
- 结果可展示（成功文本/失败原因）
- 日志可见 `transcribe ok`

- Android 版本：
- App 版本（包名/版本号/安装时间）：
- 模型配置：
- 引擎模式（auto/stub/real）：

## 2. 状态流转核对

每个任务至少核对一次：

- [ ] pending
- [ ] processing
- [ ] completed（有 result_text）
- [ ] failed（有 error_message）

异常核对：

- [ ] failed 任务可重试
- [ ] 重试期间按钮禁用，避免重复触发
- [ ] 重试完成后列表刷新正确

## 4. 结论门槛（是否通过）

通过条件（全部满足才算 PASS）：

- [ ] R1~R6 全部 PASS
- [ ] 无闪退
- [ ] 任务状态无异常卡死
- [ ] 成功/失败结果都可展示
- [ ] 至少一轮 real 模式日志包含 `transcribe ok`

最终结论：

- [ ] 本轮回归通过，可作为“重构完成度”证据
- [x] 本轮回归未通过，阻塞项见下

状态：DONE_WITH_CONCERNS（R6 未通过，属于关键阻塞）

阻塞项：

1. R6 来电/系统中断恢复提示未出现，用户感知链路不完整（P0）
2. R2/R3/R4 尚未完成并填表，缺少完整矩阵证据（P1）
3. 状态流转核对项尚未全部勾选，缺少最终留证（P1）

## 已知限制（当前版本）

- 录音中触发系统中断后，当前版本未稳定出现“自动停止并保存”提示。
- 该问题不会阻断基础录音与转写主链路，但会影响中断恢复体验一致性。
